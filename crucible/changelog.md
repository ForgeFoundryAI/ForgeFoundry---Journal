# Crucible Changelog

This file tracks major updates to the Crucible engine — the hidden rocket engine powering ForgeFoundry.

---

### v9 — April 24, 2026
- Added clear separation between **Discovery**, **Evolution**, and **Completion** layers
- Strengthened **Evolution Layer** with bolt-on Feedback Assistant (diagnostic questions + ready-to-post messages + step-by-step community guide)
- Positioned **Forge** as the main evolution/pay-point tier
- Positioned **Foundry** as the lighter production/completion layer with Debug & Test Sandbox
- Adopted rocket-engine metaphor for the overall system

### v8 — April 24, 2026
- Added diagnostic Evolution Layer (identifies weakest parameters and asks targeted questions)
- Introduced Feedback Assistant to help gather real community input
- Improved honesty and critical thinking (no "yes-man" behavior)

### v7 — April 20, 2026
- Introduced three-layer structure (Discovery, Evolution, Completion)
- Added weighted 0–100.0 scoring system
- Added news/trend enrichment capability

### v6 — April 20, 2026
- Moved from 1–10 to 0–100.0 scoring for better nuance
- Added weighted parameters

### v1–v5 — April 18–20, 2026
- Initial architecture and scoring system development
- Basic autonomous scanning and tier routing

---

**Goal:** Continuously improve Crucible by "building the machine while using the machine." Every major refinement made while developing products (Temper, BidGuard, etc.) gets documented and folded back in.
